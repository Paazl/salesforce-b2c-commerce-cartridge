Paazl SFCC cartridge : Business Manager setup

Created by Thomas Frot, last modified on mrt 22, 2019


Set up the cartridge in the Paazl Customer Area and in the Business Manager.

Before you begin

Make sure that you have:

  • Created a test account with Paazl. When it's done connect to your account (https://staging.paazl.com/login.jsp) to get your API key and API secret. You will find them under Settings > Account > Sell API
  • Finished the metadata import.

Step 1: Set up the cartridge path

To set up the cartridge path in the Business Manager: 

  1. Go to Administration > Sites > Manage Sites > [yourSiteId] > Settings. 

  2. In the Cartridges box: 
    • If you are using SFRA, enter:
      int_paazl_sfra:int_paazl_core:app_storefront_base

    • If you are using a controller-based SiteGenesis, enter:
      int_paazl_controllers:int_paazl_core:app_storefront_controllers:app_storefront_core

Step 2: Set up the Site Preferences


  1. In the Business Manager, go to Merchant tools > Site Preferences > Custom Preferences > Paazl.

  2. In the Paazl API Key box, enter the API key you got from your Paazl account. Enter the correct API key depending on if you want to connect to the test or live Paazl platform 
  3. In the Paazl API secret box, enter the API secret you got from your Paazl account. Enter the correct API secret depending if you want to connect to the test or live Paazl platform 
  4. In the Paazl Widget Endpoint box, enter the following information: 
    • On your test account:  https://widget-acc.paazl.com/v1/checkout.js
    • On your live account:  https://widget.paazl.com/v1/checkout.js
  5. Set Enable Paazl to Yes. Else the added Paazl cartridge and configuration won't be used.
  6. From the paazlWidgetAvailableTabs multi choices list, you can set what type of delivery you want to display on the Widget, DELIVERY or PICKUP or both.
  7. From the paazlWidgetDefaultTabs drop-down list, select what tab DELIVERY or PICKUP should appear first on the widget.
  8. paazlWidgetShippingOptionsLimit allow you to define how many shipping options are displayed on the widget's "Delivery" tab. The default value is 10.
  9. paazlWidgetPickupLocationsLimit allow you to define how many shipping options are displayed on the widget's "pickup" and "store" tab. The default value is 20.
  10. paazlWidgetInitialPickupLocationsLimit allow you to define the number of pickup locations initially displayed on the widget's "Pickup" and "Store" tabs
  11. paazlWidgetPickupLocationsPageLimit allow you to define the number of pickup locations displayed on the pickup location map. The default value is 10.
  12. paazlWidgetNominatedDateEnabled If true, shipping options will be displayed grouped by date. If false, the list will be displayed in ascending order of price, i.e. the least expensive option will be at the top.
  13. paazlWidgetPredefinedStyle allow you to select among one of the 6 predefine widget styles (Default, green, light green, brown, blue, red) for your widget. A seventh option call 'Customized' is available from this drop-down list. When selected, it allows you to customized the Widget style using the next paazlWidgetCustomizedStyle  attribute.

  14. paazlWidgetCustomizedStyle allows you to further customized the Paazl Widget. Below is an example of how to set it up. See Paazl Documentation for more information on how to set this site preferences.

    :root {
/* Global */
--brand-global-radius: 10px;
/* Colors */
--brand-color-background: #FFF;
--brand-color-secondary: #FFF;
/* Fonts */
--brand-font-family-title: "Futura Std", serif;
/* Title */
--brand-title-weight: 700;
/* Input */
--brand-input-border-width: 2px;
/* Button */
--brand-button-radius: 10px;
}

  15. paazlDefaultShippingOption allows you to set a fallback shipping option per country to use in case Paazl isn't responding. The format is JSON and the syntax should look like this:

    {  
   "NL":{  
      "carrier":"PostNL",
      "name":"AVG Home delivery",
      "cost":{"EUR":12.45, "USD": 14.20}
   },
  "BE":{  
      "carrier":"DHL",
      "name":"Express delivery"
   },
   "*":{  
      "carrier":"UPS",
      "name":"UPS_SAVER"
   }
}
  16. paazlDefaultSenderAddress - This is the address of the sender from which shipment will be sent. This address will be different for each client. It should be configured in preferences in below format
{
  "name": "Paazl",
  "address": {
    "city": "Amsterdam",
    "country": "NL",
    "postalCode": "1018 LL",
    "province": "",
    "street": "Jacob Bontiusplaats",
    "houseNumber": 9,
    "houseNumberExtension": ""
  }
}

17. Click on save

Step 3: Activate Paazl shipping method

  1. Go to [yourSiteId] > Merchant Tools  > Ordering >  Shipping Methods.
  2. Edit the previously imported "paazl" shipping method.
  3. If not activated yet, activate it.
  4. Click Apply.


Step 4: Configure web services endpoints

  1. Go to Administration > Operations > Services.
  2. Go to the "Credentials" tab 
  3. Select "credentials.paazl.soap" and set the URL field:
    1. On your test account:  https://staging.paazl.com/parcelshipperservice/services
    2. On your live account:  https://paazl.com/parcelshipperservice/services
    3. Click Apply.
  4. Select "credentials.paazl.rest.token" and set the URL field:
    1. On your test account:  https://api-acc.paazl.com/v1/checkout/token
    2. On your live account:  https://api.paazl.com/v1/checkout/token
    3. Click Apply.
  5. Select "credentials.paazl.rest.checkout" and set the URL field:
    1. On your test account:  https://api-acc.paazl.com/v1/checkout
    2. On your live account:  https://api.paazl.com/v1/checkout
    3. Click Apply
  6. Select "credentials.paazl.rest.commitorder" and set the URL field:
    1. On your test account:  https://api-acc.paazl.com/v1/order
    2. On your live account:  https://api.paazl.com/v1/order
    3. Click Apply.

Step 5: Configure Jobs

  1. Go to Administration > Operations > Services.
  2. Go to Paazl-Commit-Order Job
  3. Make sure the job is enabled & configured to run at frequent intervals.
  4. Make sure the job is configured to run for all sites in Business Manager for which Paazl is going to be enabled.
