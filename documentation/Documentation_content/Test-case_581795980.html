Paazl SFCC cartridge : Test case

Created by Thomas Frot, last modified on jan 09, 2019


Go to your Site storefront, add one or more products to your basket, and go to the shipping step of the checkout.

If you have follow all the previous Installation and configuration steps, Paazl should already be enabled and the paazl shipping method activated.

If this is the case, when selected Paazl as a shipping option (if not selected by default), you should see the Paazl widget below it.

If this is not the case, please verify that you have correctly followed the previous steps and try again.


  • Activate / Deactivate Paazl from your site preferences:

  1. In the Business Manager, go to Merchant tools > Site Preferences > Custom Preferences > Paazl.
  2. Set Enable Paazl to false.  
  3. Refresh or return to the shipping page of the checkout.
  4. Now if you select Paazl as shipping option, no widget should be display below it.
  5. To remove Paazl from the available shipping option, go to Merchant Tools  > Ordering >  Shipping Methods and deactivate the paazl shipping method
  6. If you refresh again the shipping page, Paazl should disappear from the list of available shipping option.
  7. After completing this test, reactivate the paazl shipping method and reset Enable Paazl to true.


  • Dynamically update Paazl available shipping option based on filled in Postcode ok geolocation Postcode (mainly for Pickup point tab):

  1. The steps below work for both Home delivery and Pickup location, but are more obvious for the pickup point options
  2. From the Shipping page, select Paazl as shipping method, the Widget should appear
  3. If not selected by default, select the Pickup tap of the widget.
  4. If the shipping form is still empty, two possibilities:
    1. SFCC geolocation is able to get a valid postcode (At the time of writing these test cases, this doesn't work for NL because the retrieved postcode geolocation is invalid: missing letters):
      1. The Pickup tap of the Widget should already show pickup locations (if any available) based on the retrieved postcode from Geolocation ()
    2. SFCC geolocation is not able to get a valid postcode (Ex: for NL because the retrieved postcode geolocation is missing the two last letters):
      1. The Pickup tap of the Widget won't show any pickup location.
  5. Start filling up the form
  6. As soon as you fill in the country and a valid postcode and exit the postcode field, the Pickup tap of the Widget should show available pickup locations (if any available for the filled in postcode).
  7. The step above only works if the filled in postcode and country match.
  8. Change the postcode and/or the postcode/country combination and verify that the available shipping option from the Paazl Widget are updated accordingly.


  • Shipping address autocompletion for Dutch addresses:

  1. This test case concerns the Dutch shipping address only
  2. Go the Shipping page if not on it already
  3. Empty the shipping form if not empty yet
  4. filled in the country, street number and postcode.
  5. When exit the postcode field, the street name and city field should be automatically completed with the correct values.


  • Change available Paazl Widget tabs:

  1. In the Business Manager, go to Merchant tools > Site Preferences > Custom Preferences > Paazl.
  2. Select one of the available values for the paazlWidgetAvailableTabs property and save.
  3. Go back to the Shipping page or refresh it if already on it.
  4. Select Paazl as shipping method.
  5. Verify that only the selected tabs are present in the Paazl Widget.
  6. Repeat the steps 1,2,3,4 and 5 with different values for paazlWidgetAvailableTabs. Possible values are DELIVERY or PICKUP or both


  • Change the default Paazl Widget tab

  1. In the Business Manager, go to Merchant tools > Site Preferences > Custom Preferences > Paazl.
  2. Set the paazlWidgetDefaultTabs to DELIVERY (make sure that Delivery is selected in the paazlWidgetAvailableTabs property) and save.
  3. Go back to the Shipping page or refresh it if already on it.
  4. Select Paazl as shipping method.
  5. Verify that the DELIVERY tab is selected by default in the Paazl Widget.
  6. Repeat the steps 1,2,3,4 and 5 with paazlWidgetAvailableTabs set to PICKUP


  • Try different Predefined Paazl Widget styles

  1. In the Business Manager, go to Merchant tools > Site Preferences > Custom Preferences > Paazl.
  2. Select one of the available values for the paazlWidgetPredefinedStyle property (except CUSTOMIZED) and save.
  3. Go back to the Shipping page or refresh it if already on it.
  4. Select Paazl as shipping method.
  5. Check that the style of the Widget has changed accordingly to the selected predefined style. This is mostly visible on the Pickup tab with the color of the Select a Different Pickup Location button.
  6. Repeat the steps 1,2,3,4 and 5 with paazlWidgetPredefinedStyle set to different value.


  • Play with the Paazl Widget style customisation

  1. In the Business Manager, go to Merchant tools > Site Preferences > Custom Preferences > Paazl.
  2. Set the paazlWidgetPredefinedStyle property to CUSTOMIZED and save.
  3. Set the paazlWidgetCustomizedStyle with different customisations and save. See Paazl Documentation about how to customise the Widget.
  4. Go back to the Shipping page or refresh it if already on it.
  5. Select Paazl as shipping method.
  6. Verify that the added customisation are visible on the Paazl Widget


  • Try different settings for the displayed shipping options limits

  1. In the Business Manager, go to Merchant tools > Site Preferences > Custom Preferences > Paazl.
  2. Set different number limits for paazlWidgetShippingOptionsLimit, paazlWidgetPickupLocationsLimit, paazlWidgetPickupLocationsPageLimit and save.
  3. Go back to the Shipping page or refresh it if already on it.
  4. See if the shipping options in the Paazl Widget on DELIVERY tab, PICKUP tab the Pickup Location map are showing up accordingly.


  • Verify shipping summary for every checkout step and confirmation page

  1. Place an order using Paazl, and check if the shipping method and shipping address (only for pickup point delivery) from the shipping summary are correct on all Checkout steps
  2. Pages to check:
    1. Billing Page
    2. Summary Page
    3. Confirmation Page


  • Check BM Order's shipment's address and custom attribute

  1. Place an order  as a registered user using Paazl.
  2. Go to [yourSiteId] > Merchant Tools  > Ordering > Orders
  3. Edit the placed order, and go to its shipment
  4. If the shipping method was a Pickup point type, check that the shipping address is the address of the pickup point.
  5. In the shipment attributes tab verify that the selected shipping method was saved under paazlSelectedShippingMethod.


  • Check Order history

  1. Create an account
  2. Log in your created account
  3. Place an order  as a registered user using Paazl.
  4. Go to [yourSiteId] > Merchant Tools  > Ordering > Orders
  5. in the storefront go to My Account page, then to Order History, find the order you have just placed and edit it
  6. Verify that its shipping method and its address (in case of pickup point delivery) information are correct.


  • Check Paazl Commit Order
  1. Place an order on the website
  2. Go to  [yourSiteId] > Merchant Tools  > Ordering > Orders and search for the order you just placed
  3. Go to the "Attributes" Tab & look for Order not yet saved/committed in Paazl attribute
  4. It should be in status "checked" which means the order is not yet committed into the paazl system.
  5. Go to Administration > Operations > Jobs and search for a job with id Paazl-Commit-Order
  6. Manually, run this job (On PIG instances this will be done automatically)
  7. Once the job is finished your order will be present in the paazl system.
  8. Check again "Attributes" Tab & now the custom attribute should be in state "checked".
